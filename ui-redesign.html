Step 01 — Heuristic Evaluation

Usability Issues Found

Evaluated against Nielsen's 10 Usability Heuristics. 8 critical issues identified across navigation, visual hierarchy, and feedback.

H1
High

No Visual Hierarchy

All text elements are nearly the same size. Users can't distinguish headings from body copy or CTAs at a glance.

→ Fix: Clear type scale + weight contrast
H2
High

Inconsistent CTA Buttons

3 different button styles used with no pattern. Some are links, some are inputs — users are confused about what's clickable.

→ Fix: Unified button component system
H3
High

Not Mobile Responsive

Layout breaks at <768px. Cards overflow, text gets clipped, navigation becomes unusable on touch devices.

→ Fix: Mobile-first grid layout
H4
Medium

Poor Colour Contrast

Grey text on white background fails WCAG AA (ratio 2.8:1). Body text is unreadable for low-vision users.

→ Fix: 4.5:1 minimum contrast ratio
H5
Medium

No System Feedback

Form submissions show no confirmation. Add-to-cart action has no visual response. Users don't know if actions worked.

→ Fix: Toast notifications + hover states
H6
Medium

Cluttered Navigation

17 navigation links with no grouping or hierarchy. Users face paradox of choice and can't find key pages quickly.

→ Fix: 5 primary links + dropdown groups
H7
Low

No Empty States

Search with no results shows a blank white page with no explanation or next-step guidance for the user.

→ Fix: Illustrated empty state + suggestions
H8
Low

Inconsistent Spacing

Padding values range from 3px to 47px with no grid. Dense sections next to airy ones feel unfinished and untrustworthy.

→ Fix: 8pt spacing grid throughout

Key Learnings

Visual Hierarchy

Type scale + weight contrast guides the eye from headline → subtext → action. Every screen needs one primary focus point.

Layout Design

An 8pt spacing grid eliminates visual noise. Consistent spacing creates rhythm — irregular spacing creates anxiety.

Responsive Design

Mobile-first forces you to prioritise. If content works at 360px, adding more for desktop is straightforward. Reverse is painful.
